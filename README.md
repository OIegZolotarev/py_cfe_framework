# Библиотека для программной генерации расширений 1С 

На основе кода из статьи - https://infostart.ru/1c/articles/1418255/

## Цели проекта

Разработать систему генерации расширений содержащих изменения в программно коде конфигурации относительно заданного коммита в системе контроля версий и текущего.
Как показывает практика - применение расширений сильно быстрей чем динамическое изменение конфигурации. С учетом того что в моей работе практикуется вносить все изменения через код - проект имеет актуальность.
Источником вдохновления стал механизм генерации патчей в СППР.

## Назначение библиотеки

Библиотека разрабатывается отдельно, для отладки механизма генерации расширений и дальнейшей её интеграции в генератор патчей.

## Ключевые шаги

- [x] Генерация почти корректных Configuration.xml и ConfigDumpInfo.xml - надо подсунуть пустую роль и язык для загрузки расширения в конфигурацию
- [x] Генерация полностью корректного пустого расширения - сгенерированное расширение загружается в конфигуратор без дополнительных манипуляций.
- [ ] Генерация расширения с модулем приложения - сгенерированное расширение загружается в конфигуратор и код из расширения выполняется корректно.
- [x] Генерация расширения с заимствованным общим модулем - расширение изменяет метод из заимствованного общего модуля.
- [ ] Генерация расширения с заимствованным модулем справочника
- [ ] И так далее...
