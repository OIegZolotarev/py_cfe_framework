# Библиотека для программной генерации расширений 1С 

На основе кода из статьи - https://infostart.ru/1c/articles/1418255/

## Цели проекта

Помочь решить две наболевшие мне проблемы:

1) Динамическое обновление в новых платформах сломано (предположительно с версии 8.3.25) - на MSSQL вылетает ошибка 8631, и на postgresql скорее всего аналогичная.
2) При работе в EDT, изменения от dev-машины до продакшена ставяться недостаточно быстро, вплоть до того что приходится дублировать работу в конфигураторе на проде и в EDT.
   Ситуацию усугубляет также сломанное динамическое обновление.   

Проблему с динамическим обновлением отчасти можно решить с помощью расширений, но сопровождать размазанные в нескольких местах версии кода становится сильно сложней.
Решением этих проблем могла бы быть система генерации патчей, по подобию того что есть в СППР и использует 1С для типовых.

Идея генерации патчей состоит в что условившись что в продакшене стоит cf собранный с определенной версии коммита, мы определяем все объекты поменявшиеся с этого коммита по текущий и генерируем cfe 
который приведет эти объекты в продакшене к версии репозитории.

## Назначение библиотеки

Библиотека разрабатывается отдельно, для отладки механизма генерации расширений и дальнейшей её интеграции в генератор патчей.
Основная цель библиотеки - генерация расширений с изменениями программного кода, остальное возможно не будет реализовано.


## Ключевые шаги

- [x] Генерация почти корректных Configuration.xml и ConfigDumpInfo.xml - надо подсунуть пустую роль и язык для загрузки расширения в конфигурацию
- [x] Генерация полностью корректного пустого расширения - сгенерированное расширение загружается в конфигуратор без дополнительных манипуляций.
- [ ] Генерация расширения с модулем приложения - сгенерированное расширение загружается в конфигуратор и код из расширения выполняется корректно.
- [x] Генерация расширения с заимствованным общим модулем - расширение изменяет метод из заимствованного общего модуля.
- [x] Генерация расширения с заимствованным модулем справочника
- [ ] И так далее...
